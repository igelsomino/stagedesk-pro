# Security policy

## Versioni supportate

La versione supportata e l'ultima release pubblicata nel repository GitHub.

| Versione | Supporto |
| --- | --- |
| Ultima release stabile | Si |
| Versioni precedenti | Solo se il problema e critico e riproducibile |

## Segnalare una vulnerabilita

Non aprire issue pubbliche per:

- credenziali, token o chiavi esposte;
- bypass di autenticazione;
- problemi che consentono accesso a dati utente o progetti locali;
- vulnerabilita negli installer, negli aggiornamenti automatici o nel deep link OAuth.

Inviare una segnalazione privata tramite GitHub Security Advisories, se disponibile nel repository.

La segnalazione dovrebbe includere:

1. descrizione del problema;
2. versione interessata;
3. sistema operativo e runtime usato;
4. passi di riproduzione;
5. impatto stimato;
6. eventuali log o screenshot privi di dati sensibili.

## Tempi di risposta

L'obiettivo e confermare la ricezione entro 5 giorni lavorativi e fornire un primo esito tecnico entro 15 giorni lavorativi, compatibilmente con gravita e riproducibilita.

## Gestione dati sensibili

Non allegare progetti teatrali reali, file media riservati, dati personali, token, chiavi private o configurazioni di produzione. Se servono evidenze, usare dati minimali o anonimizzati.
