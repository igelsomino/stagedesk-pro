# Codice di condotta

## Scopo

StageDesk Pro vuole mantenere un ambiente collaborativo rispettoso, inclusivo e orientato alla qualita del progetto.

## Comportamenti attesi

- Comunicare in modo rispettoso e costruttivo.
- Motivare proposte, bug report e revisioni con elementi verificabili.
- Accettare feedback tecnici senza attacchi personali.
- Rispettare differenze di esperienza, ruolo, lingua e contesto professionale.
- Segnalare problemi di sicurezza o dati sensibili tramite i canali indicati nella security policy.

## Comportamenti non accettati

- Insulti, molestie, discriminazioni o linguaggio offensivo.
- Pubblicazione di dati personali, credenziali, chiavi o informazioni riservate.
- Commenti deliberatamente provocatori o non pertinenti al progetto.
- Spam, abuso degli strumenti di issue o pull request, o azioni che ostacolano il lavoro degli altri.

## Applicazione

I maintainer possono moderare, modificare o chiudere issue, commenti e pull request che violano questo codice. Nei casi gravi possono limitare la partecipazione al repository.

## Segnalazioni

Per segnalazioni relative al comportamento nel progetto, aprire una issue solo se non contiene dati personali o informazioni riservate. Per problemi riservati o di sicurezza, seguire `SECURITY.md`.
